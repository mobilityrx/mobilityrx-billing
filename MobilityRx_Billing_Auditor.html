Google Drive Setup — One Time Only

1
Go to Google Cloud Console
Open console.cloud.google.com in a new tab. Sign in with your Google account (same one as your Drive).
2
Create a New Project
Click the project dropdown at the top → New Project → name it "MobilityRx Billing" → click Create. Wait a few seconds for it to create, then select it.
3
Enable the Google Drive API
In the left menu → APIs & Services → Library → search "Google Drive API" → click it → click Enable.
4
Enable the Google Picker API
Go back to Library → search "Google Picker API" → click it → click Enable.
5
Create an API Key
Left menu → APIs & Services → Credentials → Create Credentials → API Key. Copy the key that appears. Paste it in the first field below.
6
Create an OAuth 2.0 Client ID
Click Create Credentials → OAuth client ID. If prompted, configure the consent screen first (External → fill in app name "MobilityRx Billing" → save). Then: Application type = Web application. Under "Authorized JavaScript origins" add file:// and http://localhost. Click Create. Copy the Client ID (ends in .apps.googleusercontent.com). Paste it in the second field below.
7
Restrict the API Key (Recommended)
Back in Credentials, click your API key → under "API restrictions" → select "Restrict key" → check Google Drive API and Google Picker API. Click Save. This limits the key to only these two services.
8
Paste Keys Below and Click Save
Close this panel, paste both values into the fields, and click Save & Connect. You only do this once — the keys are stored locally in your browser.
🔒 HIPAA Note: Your API key is stored only in your browser's localStorage on this device. No billing data is sent to any server — files are loaded directly from Google Drive into your browser memory and processed entirely locally. The Google Picker API only shows you a file browser; it does not transmit your file contents anywhere except to this page.
